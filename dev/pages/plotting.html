Plotting

GoogleMapsAPI.jl ships two plotting paths:

  1. mapplot — a one-liner that fetches a Static-Maps background and overlays markers/polylines for any response. Lives in the Makie extension.
  2. GeoInterface trait conformance — the wrapper types (GMPoint, GMLineString, GMFeature, GMFeatureCollection) implement the GeoInterface traits, so any GeoInterface-aware plotting tool (notably GeoMakie.jl) can consume GoogleMapsAPI responses directly — with real projections, base maps, legends, and all the usual Makie layout machinery.

The built-in mapplot

Load Makie (plus a backend and FileIO) alongside GoogleMapsAPI, and mapplot(x) will fetch a static map sized to the response’s bounding box and overlay everything it can extract.

using GoogleMapsAPI
using CairoMakie, FileIO, Makie

r = geocode("1600 Amphitheatre Pkwy, Mountain View, CA")
mapplot(r)

route = compute_routes(
    (35.9132, -79.0822), (38.9805, -77.0915);
    routing_preference = "TRAFFIC_AWARE_OPTIMAL",
)
mapplot(route; maptype = "satellite", size = (800, 600))

elevations = elevation([(38.5, -120.2), (40.7, -120.95), (43.25, -126.45)])
mapplot(elevations)

Styling kwargs

Kwarg Default Purpose
maptype "roadmap" "roadmap", "satellite", "terrain", "hybrid"
size (640, 640) Image pixel dimensions
padding 0.15 Fraction of bbox span added as margin
markercolor :red Scatter colour for points
markersize 12 Scatter size
linecolor :dodgerblue Polyline colour
linewidth 3 Polyline width

Everything else — scale, language, region, style, client, key, timeout, etc. — is forwarded to static_map.

How it works

Internally, mapplot:

  1. Calls features(x) → extracts every GMPoint and GMLineString.
  2. Computes a bounding box and a Web-Mercator-fit zoom level.
  3. Fetches the Static Maps image via static_map(...) and decodes the PNG bytes with FileIO.
  4. Draws the image, then scatters markers and draws polylines at their projected pixel coordinates — so overlays line up with the real map.

Because it goes through features(x), any response type works — even a raw JSON.Object you built yourself.

GeoInterface: use responses anywhere

The real integration happens one level up. The four wrapper types (GMPoint, GMLineString, GMFeature, GMFeatureCollection) implement the GeoInterface traits, so every GeoInterface-aware package sees them as native geometries.

Traits

using GoogleMapsAPI, GeoInterface

p = GMPoint(37.4, -122.1)
GeoInterface.isgeometry(p)        # true
GeoInterface.geomtrait(p)         # PointTrait()
GeoInterface.x(p)                 # -122.1  (longitude, GeoInterface convention)
GeoInterface.y(p)                 # 37.4    (latitude)
GeoInterface.ncoord(p)            # 2
ls = GMLineString([GMPoint(0.0, 0.0), GMPoint(1.0, 2.0), GMPoint(3.0, 4.0)])
GeoInterface.geomtrait(ls)        # LineStringTrait()
GeoInterface.ngeom(ls)            # 3
GeoInterface.getgeom(ls, 2)       # GMPoint(1.0, 2.0)
fc = features(geocode("..."))
GeoInterface.isfeaturecollection(fc)           # true
f  = GeoInterface.getfeature(fc, 1)
GeoInterface.geometry(f)                       # GMPoint(...)
GeoInterface.properties(f).formatted_address   # "..."

What this unlocks

Anything in the JuliaGeo ecosystem that accepts GeoInterface geometries now accepts GoogleMapsAPI responses — including:

  • GeoMakie.jl — geographic plotting with map projections (see the next section).
  • GeoJSON.jl — serialise a feature collection to GeoJSON.
  • ArchGDAL.jl — reproject, transform, or export to shapefile/GeoPackage/etc.
  • GeometryOps.jl — distance, area, simplification, clipping.
  • Shapefile.jl, Tables.jl-backed workflows, etc.

GeoMakie integration

GeoMakie plots lat/lng data on a GeoAxis that handles projections. Because GMPoint implements PointTrait, you can pull points out of any response and hand them straight to Makie — and with a small shim, hand the wrappers themselves to GeoMakie.

Scatter: extract points, plot directly

using GoogleMapsAPI, GeoMakie, GLMakie

fig = Figure(size = (900, 600))
ax = GeoAxis(fig[1, 1]; dest = "+proj=natearth")

# Any call that returns points — geocode, places_nearby, snap_to_roads, …
r  = places("coffee"; location = (37.7749, -122.4194), radius = 2000)
pts = points(r)
lons = [GeoInterface.x(p) for p in pts]
lats = [GeoInterface.y(p) for p in pts]

scatter!(ax, lons, lats; color = :red, markersize = 10)

save("coffee.png", fig)

Routes: lines on a projected axis

using GoogleMapsAPI, GeoMakie, GLMakie, GeoInterface

fig = Figure(size = (900, 600))
ax  = GeoAxis(fig[1, 1]; dest = "+proj=wintri")

route = compute_routes(
    (35.9132, -79.0822),            # Carrboro, NC
    (38.9805, -77.0915);            # Bethesda, MD
)

for ls in linestrings(route)
    lons = [GeoInterface.x(pt) for pt in ls]
    lats = [GeoInterface.y(pt) for pt in ls]
    lines!(ax, lons, lats; color = :dodgerblue, linewidth = 3)
end

save("route.png", fig)

One function for anything

If you want to treat a response generically — plotting every point or every line without caring about the endpoint — walk the feature collection:

function geoplot!(ax, x)
    for f in features(x)
        g = GeoInterface.geometry(f)
        if g isa GMPoint
            scatter!(ax, [GeoInterface.x(g)], [GeoInterface.y(g)])
        elseif g isa GMLineString
            lons = [GeoInterface.x(pt) for pt in g]
            lats = [GeoInterface.y(pt) for pt in g]
            lines!(ax, lons, lats)
        end
    end
    ax
end

fig = Figure()
ax  = GeoAxis(fig[1, 1]; dest = "+proj=natearth")
geoplot!(ax, geocode("San Francisco, CA"))
geoplot!(ax, compute_routes((37.77, -122.42), (40.71, -74.01)))
geoplot!(ax, elevation([(40, -120), (42, -122), (44, -124)]))
fig

When to use what

Goal Use
Quick look at a response on a real base map mapplot(x)
Publication-quality projection / custom styling GeoMakie + wrappers
Hand data to another geospatial tool (GeoJSON, GDAL, …) features(x) or points(x) / linestrings(x)

mapplot is great for exploration; GeoMakie + GeoInterface is the right path when you want to compose with other layers, apply a projection, or integrate a response into a larger visualisation.
